# Self Review Checklist

Before finishing a task, review the work from five perspectives.

## Engineer

- Is the code simple?
- Is it maintainable?
- Are names clear?
- Are responsibilities separated?
- Did I remove dead code?

## Product Designer

- Is the user flow obvious?
- Is there a clear hierarchy?
- Does the UI feel polished?
- Are empty, loading, and error states handled?

## QA Engineer

- Did I test the main path?
- Did I test edge cases?
- Could this break existing behavior?
- Are errors handled?

## Performance Engineer

- Is there unnecessary work?
- Are expensive operations repeated?
- Could this slow down startup, interaction, or rendering?

## Security Engineer

- Are secrets protected?
- Is input validated?
- Are permissions minimal?
- Are external calls safe?

## Final Question

Would I confidently ship this?

If not, improve it.
