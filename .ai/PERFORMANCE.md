# Performance Standards

## Performance Is Product Quality

Slow software feels broken.

Performance should be considered during implementation, not only after users complain.

## Measure When Possible

Do not guess performance.

Measure:

- build time
- startup time
- response time
- rendering cost
- memory usage
- network latency
- bundle size

## Avoid Waste

Avoid unnecessary:

- re-renders
- polling
- network calls
- large dependencies
- blocking operations
- repeated expensive computations

## UI Performance

For UI work, check:

- scroll smoothness
- animation smoothness
- layout stability
- loading responsiveness
- interaction latency

## Practical Optimization

Do not prematurely optimize obscure paths.

Focus on user-visible performance and known bottlenecks.
